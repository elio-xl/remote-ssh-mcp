# SSH Config Editor — CRUD 操作界面设计文档

> **版本**: 1.0.0  

## 1. 目标

为 SSH Config Editor 增加配置 CRUD 操作界面，用于创建、查看、编辑和删除 SSH 连接配置。

界面需要覆盖以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `host` | string | 是 | SSH 配置别名，对应 `Host`。 |
| `hostname` | string | 是 | 目标主机地址，对应 `HostName`，可填写 IP 或域名。 |
| `user` | string | 是 | SSH 登录用户名，对应 `User`。 |
| `port` | number | 是 | SSH 端口，对应 `Port`，默认值为 `22`。 |
| `type` | enum | 是 | 连接类型。可选值：密码连接、密钥连接。 |
| `IdentityFile` | string | 条件必填 | 私钥文件路径。选择密钥连接时必填。 |
| `password` | string | 条件必填 | 登录密码。选择密码连接时必填。 |
| `workdir` | string | 否 | 连接后的默认工作目录。 |
| `remarks` | string | 否 | 备注信息，用于记录配置说明。 |

## 2. 连接类型

`type` 表示当前 SSH 连接使用的认证方式，只允许两个选项：

| 显示值 | 含义 | 条件规则 |
|--------|------|----------|
| 密码连接 | 使用用户名和密码连接 SSH | `password` 必须填写，`IdentityFile` 可为空。 |
| 密钥连接 | 使用私钥文件连接 SSH | `IdentityFile` 必须填写，`password` 可为空。 |

前端建议使用 `Select` 或 `RadioGroup` 展示连接类型，避免用户输入非法值。

## 3. 表单设计

### 3.1 添加配置

添加弹窗或页面表单包含以下字段：

```text
Host *           输入框
HostName *       输入框
User *           输入框，默认 root
Port *           数字输入框，默认 22
Type *           下拉框/单选框：密码连接、密钥连接
IdentityFile     输入框，选择密钥连接时必填
Password         密码输入框，选择密码连接时必填
Workdir          输入框，连接后的默认工作目录
Remarks          文本输入框/多行文本框
Test Connection  按钮，位于表单最下方，用于测试当前填写参数是否可连接
```

提交前必须进行表单校验：

1. `host` 不能为空。
2. `hostname` 不能为空。
3. `user` 不能为空。
4. `port` 必须是 `1-65535` 之间的整数。
5. `type` 必须是密码连接或密钥连接。
6. 当 `type` 为密码连接时，`password` 必须填写。
7. 当 `type` 为密钥连接时，`IdentityFile` 必须填写。

表单最下方需要增加 `测试连接` 按钮，位置在字段区域下方、保存/取消操作区上方。

点击 `测试连接` 时：

1. 使用当前表单中已填写但尚未保存的参数发起连接测试。
2. 连接测试前必须先执行基础表单校验。
3. 校验失败时不发起测试，并在对应字段展示错误提示。
4. 测试过程中按钮进入 loading 状态，并禁止重复点击。
5. 测试成功时显示成功提示，例如 `连接测试成功`。
6. 测试失败时显示失败原因，例如 `连接测试失败：认证失败` 或 `连接测试失败：连接超时`。
7. 测试连接不应保存配置，也不应创建、更新或删除 SSH config 条目。

### 3.2 编辑配置

编辑时需要对已有配置进行参数回填，表单字段与添加配置保持一致。

回填规则：

| 字段 | 回填来源 |
|------|----------|
| `host` | 当前选中的 SSH 配置别名，即 `Host`。 |
| `hostname` | 已有配置中的 `HostName`。 |
| `user` | 已有配置中的 `User`，不存在时默认 `root`。 |
| `port` | 已有配置中的 `Port`，不存在时默认 `22`。 |
| `type` | 根据已有配置推断：存在 `IdentityFile` 时回填为密钥连接；不存在 `IdentityFile` 但存在密码凭据时回填为密码连接；无法判断时默认密码连接并提示用户确认。 |
| `IdentityFile` | 已有配置中的 `IdentityFile`。 |
| `password` | 如果系统允许读取已保存密码，则回填；如果不允许读取明文密码，则保持为空，但在密码连接保存时要求用户重新填写。 |
| `workdir` | 已有配置中的工作目录字段，不存在时为空。 |
| `remarks` | 已有配置中的备注字段。 |

编辑保存时同样执行添加配置的全部校验规则。

编辑表单最下方同样需要提供 `测试连接` 按钮。点击时使用编辑表单中的当前值进行测试，包括已修改但尚未保存的字段。

如果允许修改 `host`，需要明确处理重命名逻辑：

1. 用户修改 `host` 时，保存操作应视为重命名配置。
2. 重命名后其它字段仍需更新到新的 `host` 条目上。
3. 如果不支持重命名，应将 `host` 字段设为只读。

### 3.3 查看配置

列表展示建议字段：

| 字段 | 说明 |
|------|------|
| `host` | 配置别名，作为主显示字段。 |
| `hostname` | 目标地址。 |
| `user` | 登录用户名。 |
| `port` | SSH 端口。 |
| `type` | 密码连接或密钥连接。 |
| `workdir` | 默认工作目录，可在列表中截断展示。 |
| `remarks` | 备注，可在列表中截断展示。 |

详情区域展示当前条目的完整参数，至少包含本需求中的 9 个字段。

### 3.4 删除配置

删除前需要二次确认，确认内容至少包含：

```text
确认删除 Host "<host>" 吗？
删除后该 SSH 配置将不可用。
```

确认后执行删除操作，并刷新列表。

## 4. 字段交互规则

### 4.1 类型切换

当用户选择密码连接：

1. 显示 `password` 字段并标记为必填。
2. `IdentityFile` 字段可以隐藏、禁用或保持可选。
3. 如果用户之前填写过 `IdentityFile`，切换到密码连接时不强制清空，但提交参数时应以密码连接规则为准。

当用户选择密钥连接：

1. 显示 `IdentityFile` 字段并标记为必填。
2. `password` 字段可以隐藏、禁用或保持可选。
3. 如果用户之前填写过 `password`，切换到密钥连接时不强制清空，但提交参数时应以密钥连接规则为准。

### 4.2 错误提示

错误提示需要明确指出缺失字段：

| 场景 | 提示 |
|------|------|
| 未填 `host` | `Host 为必填项` |
| 未填 `hostname` | `HostName 为必填项` |
| 未填 `user` | `User 为必填项` |
| `port` 非法 | `Port 必须是 1-65535 之间的整数` |
| 未选择 `type` | `请选择连接类型` |
| 密码连接未填 `password` | `密码连接必须填写 Password` |
| 密钥连接未填 `IdentityFile` | `密钥连接必须填写 IdentityFile` |

## 5. 数据提交约定

提交给后端或 API 的数据结构建议保持字段名稳定：

```json
{
  "host": "dev-server",
  "hostname": "192.168.1.100",
  "user": "root",
  "port": 22,
  "type": "password",
  "IdentityFile": "",
  "password": "example-password",
  "workdir": "/var/www/app",
  "remarks": "开发环境服务器"
}
```

`type` 内部值建议使用英文枚举，展示层再映射为中文：

| 内部值 | 展示值 |
|--------|--------|
| `password` | 密码连接 |
| `key` | 密钥连接 |

密钥连接提交示例：

```json
{
  "host": "prod-server",
  "hostname": "10.0.0.10",
  "user": "ubuntu",
  "port": 22,
  "type": "key",
  "IdentityFile": "~/.ssh/prod.pem",
  "password": "",
  "workdir": "/srv/prod",
  "remarks": "生产环境服务器"
}
```
